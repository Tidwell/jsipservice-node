Overview

The ipService library provides a single method: .get. The get method accepts a callback function as the only argument, which is passed an object containing the user's IP address. 

{
  ip: "00.000.000.000"
}

Example

Two, simple steps

1. Include the library via script tag.

<script src="http://www.jsIpService.com/ipservice.min.js"></script>

2. call .get.

function callback(user) {
  alert(user.ip);
}
ipService.get(callback);
View example on jsfiddle

With jQuery

You don't need to include our library.

Call .getJSON

$.getJSON("http://www.jsipservice.com/REST?callback=?",function(obj) {
    alert(obj.ip)
});
View example on jsfiddle

Source

The source for the library, unminified, is available at: http://www.jsipservice.com/ipservice.js or via github, along with the source for the entire site and service.

Internals

Internally, the js library caches your callback before injecting a script tag into the page, then on response, executes your callback and performs cleanup (removing the injected script tag) and removing the stored callback.

While the browser is loading the jsonP response from the ipService, the callbacks are exposed via ipService.callbacks. ipService.callbacks also has a default property that will be called if no callback is passed to get.

This default callback simply logs the ip address to console. You could overwrite ipService.callbacks.default if you would like to call .get() without passing a callback.

If the warning: 'Overwriting existing ipService variable' shows up in console, either you are including the script tag multiple times, or you are already using the ipService variable in your own code in global scope. If this is the case, feel free to download and rehost a copy of the library with whatever changes you need.

On the backend, both this website and the REST service are running through the same single-entry-point, with calls to the api being handled through a rest class, modified from this example.
